## Копирование материалов Портала в ПО Obsidian

1. Установить ПО Obsidian ([скачать](https://10.200.0.215/index.php/s/Wm6ERT8J7iZg3yR)).
1. Установить ПО GitBash ([скачать](https://10.200.0.215/index.php/s/d8aTMe7BJX8jXt2)).
1. Запустить ПО GitBash и в командном интерпретаторе ввести следующую команду:

    ```
    git clone http://10.200.0.210/msk-admin/InfoWiki.git
    ```
    
    ![](image_README/Снимок10.PNG)
    
1. Для обновления ранее скаченного репозитория необходимо перейти в каталог (с использованием командного интерпретатора), в котором он хранится (название каталога - InfoWiki, команда - __**cd InfoWiki**__), а затем ввести в командном интерпретаторе следующую команду:

    ```
    git pull
    ```
    
    ![](image_README/Снимок11.PNG)
 
1. Запустить ПО Obsidian и  открыть папку InfoWiki как хранилище.

    ![](image_README/Снимок13.PNG)
    
1. Для Вашего удобства просмотр статей необходимо проводить в режиме "Просмотр".

    ![](image_README/Снимок33.PNG)


## Авторизация пользователей на Портале

Далее рассмотрим следующий функционал Портала:
    
- сообщение администратору Портала о найденных Вами ошибках / неточностях в статьях;
- добавление Вами новых статей на Портал.

Для этого требуется пройти процедуру аутентификации на Портале. Перечень логинов и парольных фраз необходимо [скачать](https://10.200.0.215/index.php/s/XSD9ezMEfHGM6mG).
    
**Важно!!!** Требуется поменять пароль для своей учетной записи после первого захода на Портал. 

## Сообщение администратору Портала о найденных ошибках / неточностях в статьях

После прохождения процедуры аутентификации необходимо проделать следующие действия.

1. Перейти во вкладку Issues.

    ![](image_README/Снимок34.PNG)
    
1. Создать новый запрос (New issue).
    
    ![](image_README/Снимок35.PNG)

1. Заполнить поля запроса (Issue):

- Title (содержит краткое название запроса);
- Description (содержит подробное описание выявленной Вами ошибки / неточности с **ОБЯЗАТЕЛЬНЫМ** указанием ссылки на статью);
- Assignee - msk-admin (**ВАЖНО!!!** поле Assignee всегда содержит в себе значение msk-admin. Только так запрос дойдет до администратора Портала).

    ![](image_README/Снимок36.PNG)

## Добавление новых статей на Портал

После прохождения процедуры аутентификации необходимо проделать следующие действия.

1. Создать новую ветку (new branch).

    ![](image_README/Снимок24.PNG)
    
1. Дать название созданной ветки в формате: "[ваш_логин]_branch" (например, Omsk_branch).

    ![](image_README/Снимок25.PNG)
    
1. Перейти в раздел, который соответствует тематике Вашей статьи и прикрепить ее (загрузить).

    **Важно!!!** Файл с материалами должен быть в формате Markdown (.md).  

    ![](image_README/Снимок26.PNG)
    ![](image_README/Снимок27.PNG)

1.  После добавления статьи необходимо создать запрос на слияние (Merge request) с основным репозиторием Информационного портала Wiki.

    ![](image_README/Снимок30.PNG)
    
    Параметр Assignee — *msk-admin*. Кроме, того необходимо указывать __название__ и __краткое__ описание Вашей статьи.

    ![](image_README/Снимок31.PNG)
    
1.  Администратор Портала проверяет наполнение статьи и принимает / отклоняет (указав комментарии с недостатками статьи) запрос на слияние.


**ВАЖНО!!!** Другие способы загрузки новых статей использовать ЗАПРЕЩАЕТСЯ.
